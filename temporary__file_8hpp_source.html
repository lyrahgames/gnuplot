58  virtual ~temporary_file() noexcept {
59  // On Windows, the removal of the file handle is asynchronous and a file
60  // cannot be removed if there is an existing file handle to it. Therefore we
61  // may not be able to remove the file directly after closing the stream.
62  // As a consequence, we check for the removal multiple times until it
63  // succeeds by using the nothrow variant of 'filesystem::remove'.
64  // Further improvements can be added by using async operations to not block
65  // the execution of the program or by using sleep timers to not go into a
66  // busy waiting loop. But typically, this procedure is called at the end of
67  // the program. Hence, it seems not to be necessary.
68  stream.close();
